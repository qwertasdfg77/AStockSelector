# 常见问题

## 这个 App 是做什么的？

AStockSelector 是一个 Android 端 A 股日线策略筛选工具，用于学习、复盘和个人策略研究。它会在手机本地维护 K 线缓存，然后按预设战法筛选“今日信号”。

## 结果可以直接作为买卖建议吗？

不可以。筛选结果只代表历史日 K 数据满足某些技术条件，不代表未来涨跌，也不构成投资建议。实际交易需要自行判断风险。

## 为什么没有筛选结果？

常见原因有：

- 当天缓存没有更新到目标收盘交易日。
- 当前选择的战法太严格，多个战法同时选择时按“且”筛选。
- 成交额、年线距离、涨幅、K 线形态等阈值设置过窄。
- 数据源访问失败，导致部分股票没有最新 K 线。
- 停牌、ST、样本 K 线不足等基础过滤条件排除了股票。

## 为什么更新缓存比较慢？

手机端需要逐只股票联网读取并写入 SQLite，本身会受到网络、数据源限速、手机性能和后台策略影响。电脑上测试通常更快，是因为网络栈、CPU、磁盘和并发能力都更强。

## App 是实时行情吗？

不是。当前主要使用日 K 数据，适合收盘后复盘和筛选，不适合盘中实时交易。

## 数据从哪里来？

当前设计是新浪数据源优先，腾讯数据源备用。读取失败的股票会尝试备用源。详细说明见 [docs/data-sources.md](docs/data-sources.md)。

## GitHub 更新检测失败怎么办？

先检查手机能否访问 GitHub Raw 链接。如果网络不可用，可以直接到 [GitHub Releases](https://github.com/qwertasdfg77/AStockSelector/releases/latest) 下载 APK。

## 为什么 Release 里可能有 debug APK？

debug APK 方便开源项目快速安装测试，但不是正式发布签名包。仓库已提供正式签名 APK 的配置方式，只有项目维护者配置私有 keystore secrets 后，自动 Release 才会生成正式签名 APK。

## 缓存会占很多空间吗？

默认只保留约 270 个交易日的日 K 缓存，并会自动清理过早数据。实际占用会随股票数量、字段数量和 SQLite 索引变化，一般属于手机可接受范围。

## 可以接入券商自动交易吗？

当前项目不接入券商接口，也不提供自动下单能力。
