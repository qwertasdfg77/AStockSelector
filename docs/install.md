# 安装说明

## 下载安装包

推荐从 GitHub Releases 下载最新版 APK：

- <https://github.com/qwertasdfg77/AStockSelector/releases/latest>

下载后把 APK 发送到手机，或直接用手机浏览器打开 Release 页面下载。

## Android 手机安装

1. 打开 APK 文件。
2. 如果系统提示“禁止安装未知来源应用”，进入系统设置允许当前浏览器或文件管理器安装未知来源应用。
3. 返回安装页面，继续安装。
4. 首次打开后，按需要授予联网权限。

不同品牌手机的入口名称可能略有差异，常见名称包括：

- 安装未知应用
- 允许来自此来源的应用
- 外部来源应用安装

## 首次使用

1. 打开 App。
2. 点击“智能更新并筛选”。
3. 等待股票列表和日 K 缓存更新。
4. 更新完成后查看“今日信号”。

首次更新会比后续更新慢，因为需要补齐本地缓存。后续通常只更新最近一段 K 线。

## App 内程序更新

设置页里的“检测程序更新”会先显示当前版本、最新版本和更新说明。发现新版后，按钮会变成“下载更新”：

1. 点击“检测程序更新”。
2. 如果有新版，确认更新说明。
3. 点击“下载更新”。
4. 等待下载进度到 100%。
5. App 校验 APK 的 SHA256 和大小。
6. 校验通过后，系统安装器会自动打开。

如果下载失败或校验失败，App 会清理未完成的安装包，并显示失败原因；可以再次点击“下载更新”重试。

## 后台更新建议

如果手机系统会清理后台任务，可以在系统设置里给 App 打开：

- 后台运行
- 电池无限制
- 自启动或后台启动
- WLAN/移动数据联网权限

不同手机系统对后台任务限制不同，这些设置只能尽量减少被系统中断的概率。

## 常见安装失败

### 提示解析包错误

可能原因：

- APK 下载不完整。
- Android 系统版本低于 minSdk 26。
- 文件被网盘或聊天软件改名、压缩或损坏。

建议先在 App 内重新点击“下载更新”；仍失败时，再从 GitHub Releases 重新下载。

### 提示已安装更高版本

手机上已有更高 `versionCode` 的版本。可以先卸载旧版本再安装，但卸载可能清除本地缓存和设置。

### 安装后没有数据

需要先点击“智能更新并筛选”。如果数据源访问失败，请检查网络是否能访问对应数据源和 GitHub 更新地址。
